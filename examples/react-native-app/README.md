# react-native-app

React Native 0.85 example showing
[`@dvai-bridge/react-native`](../../packages/dvai-bridge-react-native)
with a backend selector (Llama / Foundation / CoreML / MLX / MediaPipe /
LiteRT). The selected backend's options are gated by platform — iOS-only
backends are dimmed on Android, Android-only ones on iOS.

## What it shows

- One screen, one focus: pick a backend, supply a model path, hit Start,
  then send a streaming chat completion through the official `openai`
  SDK pointed at the local server URL.
- The bridge is started bridgeless via the TurboModule registered by
  `@dvai-bridge/react-native`'s autolinking.

## Prereqs

- Node 22+ and pnpm (run `pnpm install` at the repo root once).
- For Android: JDK 21 + Android SDK with NDK r27+; `ANDROID_HOME` set.
- For iOS: macOS + Xcode 16+; develop over `ssh mac` if your laptop is
  Windows.
- The TurboModule build assumes RN >= 0.77 (Bridgeless on by default).

## Run on Android (any host with the SDK)

```bash
pnpm install --ignore-scripts
pnpm --filter react-native-app android
```

## Run on iOS (Mac only)

```bash
pnpm install --ignore-scripts
cd examples/react-native-app/ios
bundle install            # one-time, picks up the bundled Gemfile
bundle exec pod install
cd ..
pnpm --filter react-native-app ios
```

The pod install pulls in the `DVAIBridgeNative` pod via the workspace
package's autolinking config, which transitively pulls the `DVAIBridge`
umbrella pod. **MLX and Foundation backends require the SwiftPM-flavoured
build path** — see the
[React Native SDK guide](../../docs/guide/react-native-sdk.md) for the
`pod 'DVAIBridge', :path` workaround.

## Provide a model file

The screen's "Model path" field is where you point at a `.gguf` (Llama),
`.task` (MediaPipe), `.tflite` (LiteRT), `.mlpackage` (CoreML), or HF id
(MLX) on the device. The Foundation backend ignores `modelPath` because
Apple manages the model itself.

For real distribution, use `DVAIBridge.downloadModel({ url, sha256 })`
to stream the file with checksum verification — see the SDK guide.

## Smoke test

```bash
bash examples/react-native-app/smoke.sh
```

That runs:

1. `tsc --noEmit` — typechecks `App.tsx` against the workspace types.
2. `react-native bundle --platform android` — proves all imports
   resolve (catches version skews against
   `@dvai-bridge/react-native`'s `lib/`).
3. `./gradlew assembleDebug` if a JDK + Android SDK are present.
   Otherwise prints "skipping" and exits 0. Override with
   `SKIP_ANDROID_BUILD=1` to skip explicitly.

The iOS slice (`pod install` + `xcodebuild`) runs from the Mac mirror —
this script intentionally doesn't try iOS work on Windows.
